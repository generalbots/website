Word-Based Development

BASIC in Word Documents

Embed BASIC code blocks inside Word documents to define your bot's API endpoints, business logic, and data flows.

  • No IDE Required

    Write and edit API code directly in Microsoft Word or any .docx-compatible editor. No development environment setup needed — anyone who can open a Word document can build an API.

  • Business-Readable Logic

    BASIC syntax reads like plain English, so business analysts and domain experts can review, audit, and even write API logic themselves — no translation layer between requirements and implementation.

  • LLM-Assisted Code Generation

    Describe what your API should do in plain English and the LLM generates the BASIC code for you. Review, edit in Word, and deploy — no programming skills required.

  • Version Control via Documents

    Track changes using Word's built-in revision history. Share .docx files with colleagues for review. Your API source code lives where your team already works.

Rapid Deployment

From Document to API in Seconds

Upload your Word document and General Bots automatically parses, compiles, and deploys your API endpoints.

  • Instant Compilation

    Upload a .docx and your BASIC code is compiled into executable API endpoints instantly — no build pipeline, no CI/CD, no Docker containers. Save the file and the API is live.

  • Hot Reloading

    Re-upload updated documents and changes take effect immediately. Edit a single line in Word, save, and the API behavior changes in seconds — enabling rapid iteration.

  • S3-Compatible Storage

    Documents are stored in S3-compatible object storage with automatic processing. Upload via the API, via the web UI, or sync from your existing drive — files are indexed and available immediately.

  • Built-in Authentication

    APIs created in BASIC automatically inherit the platform's enterprise security: JWT tokens, Argon2 password hashing, and TLS encryption — no security code to write.

Full API Power

Complete API Capabilities

Access databases, call external services, handle authentication, and return structured data — all from BASIC in a Word file.

  • Database Connectivity

    Query SQL and NoSQL databases directly from BASIC code with built-in connection management and parameterized queries.

  • HTTP Integration

    Call external REST APIs, webhooks, and third-party services with simple BASIC HTTP functions.

  • Structured Responses

    Return JSON, XML, or custom formats with full control over headers, status codes, and response bodies.
