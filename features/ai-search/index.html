AI Search Revolutionized

AI Search That Actually Understands Your Questions

Traditional search matches keywords. General Bots AI Search understands meaning. It combines vector embeddings with large language models to read your documents, grasp context, and return precise, cited answers — not a list of links.

Vector Databases

How Vector Search Works

When you upload a document, General Bots splits it into chunks, converts each into a mathematical representation (an "embedding"), and stores it in a vector database. When a user asks a question, the system converts that question into the same mathematical space and finds the chunks closest in meaning — not just matching words.

  • Semantic Similarity, Not Keywords

    A user searching for "vacation policy" will find your document titled "PTO and Time-Off Guidelines" — even though the words don't match. Vector search captures meaning, so synonyms, paraphrases, and related concepts all surface the right content.

  • Multi-Language Understanding

    Because embeddings capture concepts rather than specific words, a question asked in Portuguese can match a document written in English. This is especially powerful for multinational teams sharing a single knowledge base.

  • Sub-200ms Retrieval at Scale

    Vector databases are optimized for approximate nearest-neighbor search. Even with millions of document chunks, General Bots returns the most relevant passages in under 200 milliseconds — fast enough for real-time conversation.

  • Automatic Chunking and Reranking

    You don't need to preprocess your documents. Upload PDFs, Word files, or plain text — the system automatically segments them into optimal-size chunks, generates embeddings, and applies a reranking model to ensure the best passages bubble to the top.

Large Language Models

The LLM Layer: From Retrieval to Understanding

Vector search finds the right passages. The LLM reads them, synthesizes the information, and generates a clear, human-quality answer. This combination — called Retrieval-Augmented Generation (RAG) — is how General Bots delivers answers that are both accurate and grounded in your actual data.

  • Intent Detection and Query Rewriting

    Before searching, the LLM analyzes the user's question to understand what they actually need. A vague question like "how does billing work?" gets rewritten into a precise query that targets your billing documentation, improving retrieval accuracy by up to 40%.

  • Multi-Document Synthesis

    When the answer spans multiple documents — say, a pricing sheet and a terms-of-service page — the LLM reads all relevant passages and composes a single, coherent response with citations. No more sending users to five different pages.

  • Hallucination Guardrails

    The LLM is constrained to answer only from retrieved documents. If the knowledge base doesn't contain the answer, the bot says so honestly rather than fabricating information — a critical requirement for customer-facing and compliance-sensitive deployments.

  • Provider Flexibility

    Choose from OpenAI GPT-4o, Anthropic Claude, Google Gemini, or self-hosted open-weight models. The orchestration layer handles model selection, failover, and cost optimization automatically — you can even route simple queries to cheaper models and complex ones to GPT-4.

Seamless Integration

Set Up in Minutes, Not Weeks

Most enterprise search projects take months. With General Bots, you upload your documents, write a few lines of BASIC, and deploy across WhatsApp, Teams, or any other channel. The platform handles chunking, embedding, indexing, and serving — you focus on the content.

  • BASIC-Driven Configuration

    Define your search behavior in a Word document: SET SEARCH MODE TO "semantic", SET MAX RESULTS TO 5, SET CONFIDENCE THRESHOLD TO 0.8. No YAML files, no JSON configs, no deployment pipelines. Save the .docx file and your search is live.

  • Supports Any Document Format

    Upload PDFs, Word documents, Excel spreadsheets, PowerPoint slides, plain text, Markdown, or even URLs. The ingestion pipeline extracts text, preserves structure (tables, headings, lists), and generates embeddings automatically.

  • Self-Hosted or Cloud — Your Choice

    Run the vector database on your own servers for complete data sovereignty, or use a certified hosting provider. Both options support the same API surface and the same BASIC configuration — switching between them requires zero code changes.

  • Scales from 100 to 10 Million Documents

    Start with a small FAQ and grow to a full enterprise knowledge base. The vector index is designed for horizontal scaling — add documents at any time without rebuilding the index or redeploying the bot.
